# E-PRIME Conversion Guide

Complete guide to converting text to E-PRIME (English Prime) - English without forms of "to be."

## What is E-PRIME?

E-PRIME removes all forms of the verb "to be":
- is, are, am
- was, were
- be, been, being

Result: More precise, dynamic, and actionable language.

## Why E-PRIME?

| Benefit | Example |
|---------|---------|
| **Precision** | "Data is accurate" → "Testing confirms accuracy" |
| **Active Voice** | "The process is complex" → "The process involves six steps" |
| **Accountability** | "Errors were made" → "The system created errors" |
| **Clarity** | "This is important" → "This matters" or "This affects deadlines" |
| **Dynamism** | "The system is fast" → "The system processes rapidly" |

## Conversion Patterns

### Pattern 1: "is/are" + Adjective

**Structure**: Subject + "is/are" + adjective

❌ Original:
```
The project is important.
The requirements are clear.
The solution is elegant.
```

✅ E-PRIME Options:

```
The project matters. / The project affects our timeline.
Requirements clarified by documentation.
The solution elegantly solves the problem.
```

### Pattern 2: "is/are" + Noun

**Structure**: Subject + "is/are" + noun

❌ Original:
```
This is a feature.
That is the issue.
The result is success.
```

✅ E-PRIME:

```
This feature enables real-time processing.
That issue prevents deployment.
The result demonstrates success.
```

### Pattern 3: "was/were" + Verb (past)

**Structure**: Subject + "was/were" + verb form

❌ Original:
```
The system was deployed yesterday.
The errors were identified quickly.
The data was corrupted.
```

✅ E-PRIME:

```
The system deployed yesterday.
Technicians identified errors quickly.
Corruption affected the data.
```

### Pattern 4: "be/been/being" + Verb

**Structure**: Modal/infinitive + "be" + verb form

❌ Original:
```
To be successful, you must plan.
The process is being optimized.
Data has been processed.
```

✅ E-PRIME:

```
Success requires planning.
Technicians optimize the process.
Processing completed for the data.
```

## Detailed Conversion Examples

### Example 1: Technical Documentation

❌ Original:
```
The system is designed to handle concurrent requests. Request processing is done 
in parallel. This is accomplished through multithreading. The architecture is 
based on a producer-consumer pattern. Performance metrics are impressive.
```

✅ E-PRIME:

```
The system handles concurrent requests through parallel processing. Multithreading 
enables this concurrency model. The architecture follows a producer-consumer 
pattern. Performance metrics demonstrate efficiency: 10,000 requests/second, 
50ms average latency.
```

### Example 2: Project Status

❌ Original:
```
The project is on track. Requirements are being finalized. The team is ready. 
Testing is being conducted. The launch date is set for next month.
```

✅ E-PRIME:

```
The project remains on schedule. Requirements finalize this week. The team 
completes preparations today. Testing validates functionality across all modules. 
Launch occurs next month.
```

### Example 3: Error Description

❌ Original:
```
An error was found. The error is in the authentication module. The issue was 
introduced in version 2.1. The problem is that sessions were not properly 
invalidated. This is a critical security vulnerability.
```

✅ E-PRIME:

```
Testing discovered an error in the authentication module (version 2.1). The 
system failed to invalidate sessions properly. This security vulnerability 
requires immediate remediation.
```

### Example 4: Feature Explanation

❌ Original:
```
This feature is useful for administrators. It is designed to streamline user 
management. The interface is intuitive. Users are able to perform batch operations. 
The functionality is powerful.
```

✅ E-PRIME:

```
Administrators benefit from streamlined user management. The intuitive interface 
enables batch operations, reducing manual tasks by 80%. Powerful functionality 
scales to support 10,000+ user accounts.
```

## Common "To Be" Phrases and Replacements

| ❌ "To Be" Phrase | ✅ E-PRIME Replacement | Context |
|------------------|----------------------|---------|
| "is a" | "functions as," "serves as," "represents" | Classification |
| "is that" | "involves," "means," "requires" | Definition |
| "is located" | "resides," "exists," "sits," specific location | Position |
| "is made of" | "contains," "comprises," specific materials | Composition |
| "is caused by" | "results from," "stems from," agent + verb | Causation |
| "is known as" | "goes by," "people call," specific name | Naming |
| "is similar to" | "resembles," "parallels," comparison | Comparison |
| "is different from" | "differs from," "diverges from," specific difference | Difference |
| "is related to" | "connects to," "affects," specific relation | Relationship |
| "is important" | "matters," "affects," "enables," specific impact | Importance |

## Challenging Cases

### Case 1: "It is..."

❌ Original:
```
It is important to understand this concept.
It is necessary that we meet the deadline.
It is possible to optimize further.
```

✅ E-PRIME:

```
Understanding this concept matters for success.
We must meet the deadline.
Further optimization remains possible.
```

### Case 2: Existential Statements

❌ Original:
```
There is a problem with the system.
There are several solutions available.
There is no clear consensus.
```

✅ E-PRIME:

```
The system has problems: latency issues, memory leaks, scaling limitations.
Several solutions exist: redesign architecture, optimize queries, upgrade hardware.
Consensus eludes us; stakeholders disagree on approach.
```

### Case 3: Identity Statements

❌ Original:
```
A database is a collection of organized data.
JSON is a data interchange format.
SQL is the language for databases.
```

✅ E-PRIME:

```
A database organizes and stores related data.
JSON enables lightweight data interchange between systems.
SQL provides a powerful language for database operations.
```

### Case 4: Passive Voice with "Be"

❌ Original:
```
The file was created by the system.
The update was applied by the administrator.
The task is being handled by the team.
```

✅ E-PRIME:

```
The system created the file.
The administrator applied the update.
The team handles the task.
```

```
The application is user-friendly. It is designed for beginners. The interface

✅ Solution:
```
The application emphasizes usability. Beginner-friendly design guides users

❌ Convert to E-PRIME:
```

```

- [ ] No "were" in document (except in code, quotes, or titles)
- [ ] No "be" in document (except in code, quotes, or titles)

## Notes and Exceptions

### When "To Be" is Acceptable

1. **Code blocks**: Keep as-is (part of actual code)
2. **Direct quotes**: Preserve original wording
3. **Document titles**: Keep original if established
4. **Titles and names**: Keep as official
5. **Emphasis on existence**: "Only one solution exists"
6. **Technical definitions**: "X is the Y protocol" (when no clear alternative)

### When E-PRIME Seems Awkward

If E-PRIME conversion feels forced, consider:
1. Restructure the entire sentence
2. Break into multiple sentences
3. Use more specific language
4. Ask: "What do I actually mean?"

The goal: Clearer, more precise language—not grammatical gymnastics.
